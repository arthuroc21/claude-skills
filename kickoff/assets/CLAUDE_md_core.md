## Working conventions

This project uses GitHub for version control. Changes are committed and pushed after every significant change — a decision made or reversed, a file added or removed, a feature completed, a fix landed — rather than batched up. Commit messages are clear and specific. Secrets, API keys, and `.env` files are never committed; `.gitignore` keeps them out, and the remote stays private unless deliberately made public.

Project state lives in `HANDOFF.md` at the repo root, maintained per the `handoff` skill: it is read at the start of a session and after any context compaction, and updated on every significant change, so the project can always be resumed from that one file.
